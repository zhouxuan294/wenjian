关于rem

  • 像素（pixel）：又称为“画素”，是图像显示的基本单位，最小的图像单元。这种最小的图像单元在能在屏幕上显示通常是单个的染色点。
    一般情况下，像素分为两种：
    • 设备像素（device independent pixels）：设备屏幕的物理像素，任何设备的物理像素的数量都是固定的。
    • CSS像素（CSS pixels）：又称为逻辑像素，是为web开发者创造的，在CSS和JavaScript中使用的抽象的层，每一个CSS声明和几乎所有的JavaScript属性都使用CSS像素，因此，实际上从来用不上设备像素，唯一的例外是screen.width/height。
      关于缩放：在桌面端，CSS的1个像素往往都是对应电脑屏幕的1个物理像素；而在手机端，由于手机屏幕尺寸的限制，缩放是经常性的操作。


（不论我们进行缩小还是放大操作，元素设置的CSS像素是始终不变的，而一个CSS像素对应多少个设备像素是根据当前的缩放比例来决定的）

  • 分辨率：影响清晰度或浓度的度量标准。是一个表示平面图象精细程度的概念，通常以横向和纵向点的数量来衡量的，表示成水平点数X垂直点数的形式。在一个固定的平面内，分辨率越高，意味着可能使用的点数越多，图像越细致。
    分辨率可以分为显示分辨率和图像分辨率两类；

    • 显示分辨率（屏幕分辨率）：屏幕图象的精密度，是指显示器所能显示的像素有多少。由于屏幕上的点、线、面都是由像素组成的，显示器可显示的像素越多，画面就越精细，同样的屏幕区域内能显示的信息也就越多，所以说分辨率是非常重要的性能指标之一。显示分辨率一定的情况下，显示屏越小图像越清晰，反之，显示屏大小固时，显示分辨率越高图像越清晰。
    • 图像分辨率：单位英寸中所包含的像素点数，其定义更趋近于分辨率本身的定义。

  • DPR:设备像素比（device pixel ratio），是默认缩放为100%的情况下，设备像素和CSS像素的比值，即物理像素与逻辑像素的比值。
    （DPR=设备像素/CSS像素）

  • rem（font size of the root element）：是指相对于根元素的字体大小的单位。例如：html 的 font-size 是15px，这 1rem=15px。移动端响应式布局中使用 rem 单位。

以iPhone6为例：

  • DPR=2；
  • 设备独立像素（CSS像素，横向）：325px；
  • 物理像素：750px；
  • 对应的设计稿的宽度：750px；
  • 假设：
    • font-size-A为对应设备独立像素的html的字体大小，
    • font-size-B为对应设计稿的html的字体大小；

为了让设计稿（其像素为CSS像素）能在iPhone6中完美展现，应该使设计稿宽度750px与font-size-B的比值等于设备独立像素325px与font-size-A的比值，即：
750 / font-siz-B = 325 / font-size-A；
仅为了方便计算，font-size-B取100px；
则：font-size-A = 325 / 750 * 100px；
此时，也就是：1rem= 325 / 750 * 100px；
之后，设计稿中的100px，在写页面的时候就是2rem了。
